# AES-256-GCM Wire Packet Protocol Specification — Project Synapse

## Cipher Configuration

| Parameter | Value |
|-----------|-------|
| Algorithm | AES-256-GCM (Authenticated Encryption with Associated Data) |
| Key size | 256 bits (32 bytes) |
| Nonce size | 96 bits (12 bytes) — generated via `secrets.token_bytes()` |
| Authentication tag | 128 bits (16 bytes) — integrated by AESGCM |
| Key source | `BCI_SHARED_KEY` environment variable (64-char hex string) |
| Library | `cryptography.hazmat.primitives.ciphers.aead.AESGCM` |

## Wire Format

```
[12-byte Nonce][Variable-length Ciphertext + 16-byte Auth Tag]
```

- **Nonce:** Random 12 bytes. MUST be unique per encryption operation to prevent nonce reuse vulnerability.
- **Ciphertext + Auth Tag:** AESGCM.encrypt() output. Authentication tag is automatically appended and verified on decrypt.
- **Total overhead:** 12 bytes (nonce) per frame. No padding.

## Frame Payload (Plaintext JSON before encryption)

### Client → Server (Headset Frame)

```json
{
    "user_id": "string",
    "sample_id": 0,
    "raw_channels": [0.0, ...],
    "game_context": [0.0, ...],
    "timestamp_us": 0
}
```

### Server → Client (Action Response)

```json
{
    "user_id": "string",
    "sample_id": 0,
    "action_id": 0,
    "confidence": 0.0,
    "p_error": 0.0
}
```

## Key Management Directive

- The 256-bit key is supplied exclusively through the `BCI_SHARED_KEY` environment variable.
- On Linux/macOS: `export BCI_SHARED_KEY=$(openssl rand -hex 32)`
- On Windows: `set BCI_SHARED_KEY=<64-hex-chars>`
- The key is read once at process startup by `SecureBCIGateway.__init__()`.
- If the variable is unset or empty, the process raises `ValueError` at startup — it will not run with a default or hardcoded key.
- No key material is written to disk, logs, or crash dumps by the runtime.

## Security Properties

| Property | Guarantee |
|----------|-----------|
| Confidentiality | AES-256 encryption prevents plaintext recovery |
| Integrity | GCM authentication tag detects any bit-level modification |
| Replay protection | Each frame uses a unique random nonce (12 bytes, `secrets.token_bytes`) |
| Key isolation | No plaintext key in source, config files, or binary — env var only |
| Forward secrecy | Not provided (no ECDH key exchange). For forward secrecy, deploy behind TLS 1.3. |

## Cryptographic Dependencies

```python
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
```

This is the only cryptographic primitive in the hot path. No libsodium, no OpenSSL direct calls, no custom crypto.
