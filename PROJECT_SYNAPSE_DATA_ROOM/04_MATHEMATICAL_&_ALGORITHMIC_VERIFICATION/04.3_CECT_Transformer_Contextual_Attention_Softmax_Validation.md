# CECT Transformer — Contextual Attention Softmax Validation

## Architecture

The Context-Aware Error Rectification Transformer is a pure-NumPy implementation of a 2-layer, 2-head, d_model=32 transformer with an integrated error-correction head.

```
Input: v_t (tangent feature vector, d=10) + ctx (game context, d=4)
          │
          ▼
    Linear projection → d_model=32
          │
          ▼
    LayerNorm → MultiHeadSelfAttention (2 heads) → Residual
          │
          ▼
    LayerNorm → FeedForward (32→64→32, ReLU) → Residual
          │
          ▼
    LayerNorm → MultiHeadSelfAttention (2 heads) → Residual
          │
          ▼
    LayerNorm → FeedForward (32→64→32, ReLU) → Residual
          │
          ▼
    Output heads:
      ├── action_logits → softmax → action_id
      ├── confidence → sigmoid → [0, 1]
      └── p_error → sigmoid → [0, 1]
```

### Attention Head Detail

```
MultiHeadSelfAttention(d_model=32, n_heads=2):
    W_q, W_k, W_v: each 32×(32/2)=32×16 per head
    head_i = softmax(Q_i · K_i^T / sqrt(16)) · V_i
    output = concat(head_1, head_2) · W_o  (W_o: 32×32)
```

The softmax temperature is fixed at `sqrt(d_k) = 4`. The entropy of the attention distribution is used as a feature in the confidence head.

## Error-Related Negativity (ERN) Tracking

The CECT model is trained to detect ERN patterns in the tangent feature stream:

- **ERN signature:** A sharp negative deflection in the fronto-central EEG channels 100–200 ms post-error, visible as a transient increase in theta-band power and a stereotyped covariance structure.
- **Modeling approach:** During training, the system injects synthetic ERN-like perturbations into the tangent vector stream. The CECT model learns to suppress action output when the tangent features resemble an ERN template.
- **Inference:** The `p_error` head outputs a probability that the current frame contains an error-related signal. When `p_error > 0.5`, the `TemporalVotingEnsemble` treats the frame as high-noise and applies additional gating.

## Training

The CECT model is trained on synthetic data generated by perturbing clean tangent features with known error patterns:

```python
# Training loop:
for step in range(500):
    # Generate synthetic tangent vectors with injected errors
    v_clean = tanh(W_context @ context)
    v_noisy = v_clean + noise_scale * error_pattern_mask
    # Train on mixed clean/noisy batches
    loss = cross_entropy(action_pred, action_true) + \
           mse(confidence_pred, 1.0 - error_flag) + \
           mse(p_error_pred, error_flag)
```

## Softmax Validation

The attention softmax is validated to produce proper probability distributions:

- Sum-to-1: `sum(softmax(z)) = 1.0 ± 1e-12`
- Monotonicity: `z_i > z_j ⇒ softmax(z)_i > softmax(z)_j`
- Shift-invariance: `softmax(z + c) = softmax(z)`
- Numerical stability: inputs clamped to [-10, 10] before softmax to prevent exp overflow
- Entropy bounds: `0 ≤ H(softmax(z)) ≤ log(d_k)` for d_k = 16
